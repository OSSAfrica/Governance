# Decision-Making Process

This document defines how OSSAfrica governance decisions are proposed,
reviewed, and approved.

## Decision Types

- Operational decisions: routine execution and day-to-day activities
- Policy decisions: governance, conduct, and process updates
- Strategic decisions: long-term direction and major initiatives

## Standard Workflow

1. Open a proposal issue or pull request.
2. Provide context, expected impact, and alternatives considered.
3. Allow feedback window from maintainers and community members.
4. Record outcome and rationale in the proposal thread.
5. Merge approved updates and reference the decision record.

## Consensus Model

- Prefer rough consensus through open discussion.
- Escalate unresolved disputes to the OSSAfrica SIG.
- Escalate governance conflicts to OpenSSF BEAR WG when required.

## Documentation Requirements

Each decision should include:

- Decision owner
- Decision date
- Affected documents
- Summary of alternatives
- Final rationale

## Review Cadence

Review this process quarterly and update as governance matures.
